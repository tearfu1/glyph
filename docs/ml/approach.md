# ML-компонент: генерация ответов в стиле автора

## Постановка задачи

Пользователь задаёт вопрос по книге. Автор недоступен (не зарегистрирован / нет в живых).
Система генерирует ответ **в стиле автора**, опираясь на его произведения.

Это не использование стороннего API — это самостоятельно обученная модель с математическими выкладками.

---

## Подход: RAG + LoRA Fine-tuning

Два столпа:
1. **RAG (Retrieval-Augmented Generation)** — находит релевантные фрагменты из произведений автора
2. **LoRA Fine-tuning** — дообучает языковую модель на стиль конкретного автора

### Почему именно так

| Подход | Плюсы | Минусы | Научность |
|--------|-------|--------|-----------|
| Только RAG + промпт | Просто, быстро | Стиль не контролируется, по сути prompt engineering | Слабая |
| Только fine-tuning | Стиль хорошо улавливается | Модель не знает контекст конкретной книги, галлюцинации | Средняя |
| **RAG + LoRA** | Релевантный контекст + стиль автора | Сложнее в реализации | **Высокая** |

## 1. RAG — Retrieval-Augmented Generation

### 1.1 Препроцессинг текстов

Произведения автора → чанки:
- Размер чанка: ~512 токенов (~300-400 слов)
- Overlap: 64 токена (перекрытие для сохранения контекста на границах)
- Метаданные: book_id, chapter, position

### 1.2 Эмбеддинги

Каждый чанк → вектор через модель-энкодер.

**Модель:** `intfloat/multilingual-e5-base` (768-мерные вектора, мультиязычная, хорошо работает с русским)

Альтернатива: `cointegrated/rubert-tiny2` (312-мерные, быстрее, но менее точная)

### 1.3 Математический аппарат

**Self-Attention (основа Transformer-энкодера):**

$$Attention(Q, K, V) = softmax\left(\frac{QK^T}{\sqrt{d_k}}\right) V$$

Где:
- $Q = XW_Q$ — матрица запросов (queries)
- $K = XW_K$ — матрица ключей (keys)
- $V = XW_V$ — матрица значений (values)
- $d_k$ — размерность ключа (для нормализации)
- $W_Q, W_K, W_V \in \mathbb{R}^{d_{model} \times d_k}$ — обучаемые проекции

**Multi-Head Attention:**

$$MultiHead(Q, K, V) = Concat(head_1, ..., head_h) W^O$$

$$head_i = Attention(QW_i^Q, KW_i^K, VW_i^V)$$

**Косинусная близость (для retrieval):**

$$sim(\mathbf{q}, \mathbf{d}) = \frac{\mathbf{q} \cdot \mathbf{d}}{||\mathbf{q}|| \cdot ||\mathbf{d}||}$$

Где $\mathbf{q}$ — вектор вопроса, $\mathbf{d}$ — вектор чанка.

**Retrieval:** выбираем top-k чанков с максимальной близостью:

$$C = \{d_i \mid d_i \in \text{top-k}(\text{sim}(\mathbf{q}, \mathbf{d}_i))\}$$

### 1.4 Векторное хранилище

**Qdrant** — vector database, написана на Rust. Поддерживает:
- HNSW индекс для approximate nearest neighbor search
- Фильтрация по метаданным (author_id, book_id)
- REST API и gRPC

Структура коллекции:
```json
{
  "id": "chunk_uuid",
  "vector": [0.123, -0.456, ...],  // 768 dims
  "payload": {
    "author_id": 1,
    "book_id": 42,
    "chapter": "Часть первая, глава III",
    "text": "Раскольников вышел из своей каморки..."
  }
}
```

## 2. LoRA — Low-Rank Adaptation

### 2.1 Идея

Вместо обновления всех весов модели (full fine-tuning), мы замораживаем исходные веса и добавляем маленькие обучаемые матрицы.

### 2.2 Математическое обоснование

Для весовой матрицы $W_0 \in \mathbb{R}^{d \times k}$ предобученной модели:

$$W = W_0 + \Delta W = W_0 + BA$$

Где:
- $B \in \mathbb{R}^{d \times r}$ — "down-projection"
- $A \in \mathbb{R}^{r \times k}$ — "up-projection"
- $r \ll \min(d, k)$ — ранг (гиперпараметр, обычно 4-16)

**Сокращение параметров:**
- Full fine-tuning: $d \times k$ параметров
- LoRA: $r \times (d + k)$ параметров

**Пример для rugpt3small** ($d = k = 768$, $r = 8$):
- Full: $768 \times 768 = 589\,824$ параметров на слой
- LoRA: $8 \times (768 + 768) = 12\,288$ параметров на слой
- **Сокращение в 48 раз**

### 2.3 Инициализация

- $A$ инициализируется случайным Гауссовым распределением
- $B$ инициализируется нулями
- При старте обучения: $\Delta W = BA = 0$ → модель начинает с предобученных весов

### 2.4 Масштабирование

$$h = W_0 x + \frac{\alpha}{r} BAx$$

Где $\alpha$ — коэффициент масштабирования (гиперпараметр). Типичное значение: $\alpha = 16$ при $r = 8$.

### 2.5 Функция потерь

Стандартная cross-entropy для авторегрессионного языкового моделирования:

$$\mathcal{L} = -\sum_{i=1}^{N} \log P(x_i \mid x_1, x_2, ..., x_{i-1}; \theta)$$

Где $\theta$ включает только параметры LoRA ($A$ и $B$), замороженные веса $W_0$ не обновляются.

### 2.6 Базовая модель

**`ai-forever/rugpt3small_based_on_gpt2`** (~125M параметров)

Или **`ai-forever/rugpt3medium_based_on_gpt2`** (~350M) если позволяет GPU.

Обе модели предобучены на большом русскоязычном корпусе (Taiga, Wikipedia, и др.).

### 2.7 Отдельный адаптер на каждого автора

Ключевое: базовая модель **одна**, но для каждого автора свой набор LoRA-весов:

```
models/
├── base/                    # rugpt3small (замороженные веса)
├── adapters/
│   ├── dostoevsky.bin       # LoRA веса (~5 MB)
│   ├── chekhov.bin          # LoRA веса (~5 MB)
│   └── bulgakov.bin         # LoRA веса (~5 MB)
```

При инференсе: загружаем базу + нужный адаптер → генерируем.

## 3. Генерация ответа

### 3.1 Промпт-шаблон

```
Контекст из произведений автора:
{chunk_1}
{chunk_2}
{chunk_3}

Вопрос читателя: {question}

Ответ в стиле автора:
```

### 3.2 Параметры генерации

| Параметр | Значение | Зачем |
|----------|---------|-------|
| `max_new_tokens` | 256-512 | Ограничение длины ответа |
| `temperature` | 0.7-0.9 | Баланс креативности/когерентности |
| `top_p` | 0.9 | Nucleus sampling |
| `repetition_penalty` | 1.1-1.3 | Избегание повторов |
| `do_sample` | True | Стохастическая генерация |

## 4. Метрики оценки

### 4.1 Качество языковой модели

**Perplexity (PPL):**

$$PPL = \exp\left(-\frac{1}{N}\sum_{i=1}^{N} \log P(x_i \mid x_{<i})\right)$$

Чем ниже — тем лучше модель предсказывает текст. Измеряем на held-out части корпуса автора.

### 4.2 Стилевые метрики

| Метрика | Формула / описание | Что показывает |
|---------|-------------------|----------------|
| **TTR** (Type-Token Ratio) | $\frac{\text{уникальные слова}}{\text{все слова}}$ | Лексическое разнообразие |
| **Средняя длина предложения** | $\frac{\text{все слова}}{\text{кол-во предложений}}$ | Синтаксический стиль |
| **Распределение частей речи** | Доля существительных, глаголов, прилагательных | Грамматический стиль |
| **Hapax legomena ratio** | $\frac{\text{слова, встреченные 1 раз}}{\text{все слова}}$ | Уникальность лексики |

Сравниваем эти метрики для:
- Оригинальных текстов автора
- Сгенерированных ответов с LoRA
- Сгенерированных ответов без LoRA (baseline)

### 4.3 Семантическое качество

**Cosine Similarity** между эмбеддингом ответа и эмбеддингами релевантных чанков:

$$\text{semantic\_score} = \frac{1}{k}\sum_{i=1}^{k} sim(\text{emb}(answer), \text{emb}(chunk_i))$$

### 4.4 BLEU (опционально)

Если есть реальные ответы авторов (например, из интервью):

$$BLEU = BP \cdot \exp\left(\sum_{n=1}^{N} w_n \log p_n\right)$$

### 4.5 Human Evaluation

Слепое тестирование: показываем 3 варианта (оригинальный текст автора, LoRA-генерация, базовая модель без LoRA) — оценщик определяет, где автор, а где AI.
